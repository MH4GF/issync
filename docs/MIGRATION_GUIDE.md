# progress-document-template.md Migration Guide

このドキュメントは、progress-document-template.mdのバージョン間の変更内容とマイグレーション手順を記録します。

## バージョン管理の方針

- **バージョン番号**: 単純な整数（1, 2, 3...）を使用
- **日付**: バージョン番号と共に日付を併記（例: `<!-- Template Version: 1 (2025-10-15) -->`）
- **マイグレーション**: このガイドを参照して、既存のplan.mdを新しいバージョンに更新

**全バージョン共通のマイグレーション最終ステップ**: 変更適用後、plan.mdのバージョンヘッダー（2行目）を対象バージョンに更新

## Version 1-6 (2025-10-15 ~ 2025-10-16)

**主な変更:**
- v1: 初期バージョン（11セクション、HTMLコメントガイダンス）
- v2: Open Questionsのカテゴリ構造をフラット化
- v3: 更新ガイドラインをHTMLコメント化
- v4: issyncマーカー追加（`<!-- issync:v1:start/end -->`）
- v5-6: TasksセクションのGitHub Issue連携ガイダンス整理

**マイグレーション**: カテゴリ見出し削除、ガイドラインHTMLコメント化、issyncマーカー追加

## マイグレーションツール

**`/compact-progress-document`コマンド:**
- バージョンヘッダー読み取り → 最新版と比較 → マイグレーション提案 → 承認後実行

**手動バージョン確認:**
- plan.md先頭行 `<!-- Template Version: X (YYYY-MM-DD) -->` を確認

**トラブルシューティング:**
- バージョンヘッダーがない: 先頭に追加し、現在の構造から最も近いバージョンを特定
- マイグレーション失敗: GitHub Issueのコメント履歴からバックアップを確認

## Version 15 (2025-11-06)

**Confidence Assessmentセクション追加（プロジェクト全体の実装確信度評価）**

**変更内容:**
- 新セクション「Confidence Assessment」をOutcomes & Retrospectivesの直後、Inboxの直前に追加
- プロジェクト全体の確信度を3段階（High 🟢/Medium 🟡/Low 🔴）で評価
- 各Phase完了時に評価を置き換えて更新（進捗の変遷を追跡）
- AIエージェント向けガイダンスコメントに評価基準と更新タイミングを明記

**理由:**
- Open Questionsの推奨案自信度（未解決の問いに対する検討案の実現可能性評価）とは異なり、プロジェクト全体の実装確信度を可視化
- AIエージェントと人間がプロジェクトの健全性を迅速に把握できるようにする
- 各Phaseでの評価変遷を記録し、リスク管理を強化

**マイグレーション手順:**

#### 手動マイグレーション

既存のplan.mdをバージョン15に移行する手順:

1. `## Outcomes & Retrospectives`セクションの直後、`## Inbox`セクションの直前に新セクションを挿入
2. 以下のテンプレートをコピー&ペースト:

**追加内容:**
```markdown
---

## Confidence Assessment

<!--
📝 Guidance for AI
記入タイミング: 各Phase完了時（poc完了後、architecture-decision完了後、implement完了後など）に必須更新
記入内容: プロジェクト全体の実装確信度を評価。各Phaseで前回の評価を置き換えて更新する

評価基準:
- High 🟢: 実装方針が確定し、技術的リスクが低く、スムーズな実装が見込まれる
- Medium 🟡: 実装方針はあるが、一部不確実性やリスクが残る
- Low 🔴: 重要な技術的決定が未解決、または実装に大きなリスクがある

**重要**: この評価はプロジェクト全体の確信度を示すもので、Open Questionsセクションの推奨案自信度（未解決の問いに対する検討案の実現可能性評価）とは異なる
-->

**Overall Confidence**: [High 🟢 / Medium 🟡 / Low 🔴] - [プロジェクト全体の確信度の理由を1行で説明]

---
```

3. 現在のPhaseに応じて初期評価を記入:
   - plan完了後: 通常Medium 🟡（仕様が固まったが実装前）
   - poc完了後: 技術検証結果に基づいて評価
   - architecture-decision完了後: 設計が固まればHigh 🟢の可能性
   - implement完了後: 実装完了ならHigh 🟢

**注意事項:**
- 既存のOpen Questions自信度レベル（🟢高/🟡中/🔴低）はそのまま維持（用途が異なる）
- 各Phase完了時に前回の評価を置き換えて更新（履歴は残さない）
- 評価理由は1行で簡潔に記述

---

## Version 16 (2025-11-06)

**Inboxセクション削除**

**変更内容:**
- Inboxセクション削除（9セクション → 8セクション）
- AIエージェント主導の開発ワークフローに最適化
- 実際にほとんど使用されていない人間専用セクションを削除し、テンプレートをシンプル化

**理由:**
- 進捗ドキュメントの大半でInboxセクションが空のまま残っている
- AIエージェントは記入せず、人間が記入する前提だが、実際の運用ではほとんど使用されていない
- AIエージェント主導の開発を推進する中で、人間専用のセクションは必要最小限にすべき

**マイグレーション手順:**
1. `## Inbox`セクション（Confidence Assessmentの直後、`<!-- issync:v1:end -->`の直前）を削除
2. 既存のplan-*.mdファイルは変更不要（新規作成時のみVersion 16テンプレートを使用）
