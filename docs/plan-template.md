**GitHub Issueとの対応:**
- **大きなタスク**（複数日、複数PR）→ サブissue化し`(#123)`形式でIssue番号を記載
- **小さなタスク**（1-2時間で完結）→ このissue内で管理、Issue番号なし
- **サブissue化検討中** → `(未Issue化)`と記載

**記法:**
- `- [ ] タスク名` - 未完了の小タスク
- `- [ ] タスク名 (#123)` - サブissueとして管理中
- `- [ ] タスク名 (未Issue化)` - サブissue化を検討中
- `- [x] タスク名` - 完了済みタスク
